## 1. Background and Scope

### 1.1 Project Background

Jotted-down inspirations often get "stored" but never truly "processed" — traditional notes don't ask follow-ups, don't diverge, and don't converge into actionable insights. Three months later, revisiting them, the original thought process has faded.

This project experiments with an on-device **Gemma-4-E2B-IT** model + LoRA fine-tuning to build an Android "AI Thinking Assistant": every quick capture goes through structured "question—diverge—converge" processing, eventually沉淀为 an actionable "inspiration card".

### 1.2 Scope of This Document

Sprint 1 Week 1's core goal is: **get data and evaluation infrastructure in place, and verify the base model can run through Layer 2 regression set** (see `sprint-1-train.md`).

This document does not cover training and PoC (that's Week 2 onwards), only a "completion checklist" of Week 1 delivered artifacts, paths, and conclusions for handoff to Week 2 and retrospective archiving.

## 4. Baseline Conclusions (For Week 2 Reference)

Judge `overall` dimension (1–100) means below (from `layer2_judge_summary.json` and `s1-baseline-report_EN.md` §5):

| Stratum | Sample Count | overall Mean |
|---------|--------------|--------------|
| core | 200 | **93.35** |
| general | 200 | **81.85** |
| zh_guard | 100 | **77.94** |
| **All** | **500** | **85.67** |

**Redline Conclusion**: P0 (safety), P1 (function) **not triggered**; P2 (experience) **warning triggered** — Chinese protection stratum (zh_guard) significantly below core stratum, Week 2 PoC must include **zh_guard** in acceptance metrics. See baseline report §6 and Sprint1-03 §7.2.
