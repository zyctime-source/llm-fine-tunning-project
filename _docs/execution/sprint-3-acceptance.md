# Sprint 3（第3个月）Demo 收敛与里程碑验收

## 目标

- 形成可演示、可复盘、可继续迭代的里程碑版本。
- 完成 Layer 3 人工验收并形成结论。

## 时间与投入

- 周期：4 周
- 预计投入：约 80 小时（每周 20 小时）
- 预算建议：120-180 GPU 小时中的 20-40 小时用于修正式微调与回归

## Must / Should / Won't

- Must
  - 聚焦稳定性问题：崩溃、超时、空输入/超长输入边界。
  - 聚焦可用体验：输出结构一致性、中文可读性、基础响应速度。
  - 执行 Layer 3 人工验收集并形成通过结论。
  - 完成里程碑归档：模型版本、评测结果、风险清单、后续 backlog。
- Should
  - 针对高频失败场景补充最小修复策略说明。
  - 形成 Demo 演示脚本（3-5 分钟固定路径）。
- Won't
  - 不新增大范围功能需求。
  - 不开启 Stage 2 个性化微调。

## 每周拆分

### Week 1：稳定性集中修复

- 统计并处理高频崩溃与超时问题。
- 补齐边界输入测试并复测。

交付物：
- `s3-stability-fix-log`
- `s3-boundary-test-report`

### Week 2：体验与可读性收敛

- 优化输出结构化一致性。
- 优化中文可读性与基本响应节奏。

交付物：
- `s3-quality-tuning-report`
- `s3-cn-readability-check`

### Week 3：Layer 3 人工验收

- 执行生产验收集走查。
- 对争议样本做人工仲裁与分类。

交付物：
- `s3-layer3-acceptance-result`
- `s3-dispute-cases`

### Week 4：里程碑归档与发布准备

- 输出里程碑验收报告（通过/有条件通过/不通过）。
- 完成下一阶段路线图和 backlog 列表。

交付物：
- `milestone-demo-v1`
- `s3-milestone-report`
- `next-stage-backlog`

## 里程碑通过标准

- 可稳定演示端到端主流程。
- 关键评测结论可复现，且不触发 P0/P1 红线。
- 现有文档可支持后续迭代，不依赖口头同步。

## 风险与止损

- 稳定性未收敛：优先保证演示路径稳定，非关键问题后移。
- 中文可读性不足：接受“可用但不完美”，记录到后续增强 backlog。
- 验收争议多：按 P0/P1/P2 分类处理，先清红线再提升体验。

## Sprint 结束复盘模板

- 里程碑结论：通过 / 有条件通过 / 不通过
- 演示成功率与失败原因
- 关键风险是否已收敛
- 下一阶段前 5 项优先任务
