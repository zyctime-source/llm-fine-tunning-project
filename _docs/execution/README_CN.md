# 三个月执行总览（Execution）

本目录用于落地“三个月 Sprint 执行计划”，聚焦可执行任务、门禁标准、风险止损与缓冲规则。

## 文件导航

- `sprint-1-train.md`：第 1 个月，训练主线打通（PoC + Stage 1）
- `sprint-2-demo.md`：第 2 个月，评测稳定化 + Android 最小主链路
- `sprint-3-acceptance.md`：第 3 个月，Demo 收敛与里程碑验收
- `buffer-sprint-policy.md`：缓冲 Sprint 触发与执行规则

## 执行顺序

1. 先执行 Sprint 1，并完成 Gate1 评审。
2. 通过 Gate1 后进入 Sprint 2，并完成 Gate2 评审。
3. 通过 Gate2 后进入 Sprint 3，完成里程碑验收。
4. 若任一门禁未通过，按 `buffer-sprint-policy.md` 进入缓冲 Sprint。

## 每周固定动作

- 周初：确认 Must / Should / Won't
- 周中：更新风险与阻塞
- 周末：输出周结（完成项、未完成项、下周调整）

## 每个 Sprint 结束固定输出

- Gate 评审结论（通过 / 有条件通过 / 未通过）
- 评测变化摘要（核心能力、中文保护题、稳定性）
- 成本摘要（GPU 小时、API 调用）
- 下一 Sprint 输入清单（3-5 条）
