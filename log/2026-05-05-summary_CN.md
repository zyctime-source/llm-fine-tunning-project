## 2. 第 8 章：训练与实验策略（Train / Iterate）· 已定要点

### 2.1 训练阶段划分（与两阶段微调对应）

| 阶段 | 目标 | 关键决策点 |
|------|------|------------|
| **阶段 0 PoC** | 验证数据配方可行性，建立基线 | 能产出可评估的 LoRA 权重 |
| **阶段 1-A 保守** | 产出可用的「脑暴 + 总结」基础模型 | Gemma-4-E2B vs Qwen3.5-2B 对决 |
| **阶段 1-B 激进** | 1-A 效果不佳时调整配方 | 防过拟合，设止损线 |
| **阶段 2 个性化** | 学习用户个人风格 | 累积 N 条定稿卡片后触发 |

### 2.2 版本命名规范

- **格式**：`{阶段}-{基座模型}-{数据版本}-{实验序号}-{状态}`
- **示例**：`s1-qwen35-2b-v1.0-e03-done`、`poc-gemma4e2-v0.1-e01-wip`
- **数据版本**：`v{主}.{次}`，主版本 = 配方重大变更，次版本 = 配比微调

### 2.3 可复现性规范

- **随机种子**：探索性实验固定 `42`，消融实验跑 3 个种子报告均值
- **环境快照**：记录框架版本、CUDA 版本、硬件规格、数据集哈希
- **实验血缘**：树状历史，每个实验标注 `parent_experiment`

### 2.4 实验决策类型

| 决策 | 含义 |
|------|------|
| **Accept** | 达到预期，进入下一阶段 |
| **Iterate** | 有改进需微调，基于本次开新实验 |
| **Reject** | 明显倒退，回退到父实验 |
| **Abandon** | 方向错误，归档不再继续 |

## 3. 第 9 章：评测与质量（Eval / QA）· 已定要点

### 3.1 题集分层体系（三层）

| 层级 | 规模 | 用途 | 更新频率 |
|------|------|------|----------|
| **Layer 1 能力探针** | ~4,000+ 条 | 全面能力探测，横向对比 | 随公开基准更新 |
| **Layer 2 回归验证** | ~500 条 | 每次实验后自动评估，检测退化 | 定期补充历史错题 |
| **Layer 3 生产验收** | ~100 条 | 最终人工走查，确认产品可用 | 极少变动 |

### 3.2 评估维度（不定公式，定框架）

- 相关性、连贯性、有用性、创造性、结构化、中文质量
- 不同题集可侧重不同维度
- 输出「分维度雷达图」，不追求单一总分

### 3.3 质量红线类型

| 级别 | 类型 | 触发示例 | 决策 |
|------|------|----------|------|
| **P0** | 安全红线 | 有害输出、隐私泄露 | **立即停止**，排查修复 |
| **P1** | 功能红线 | 核心能力退化 > 20% | **拒绝实验**，回退基座 |
| **P2** | 体验红线 | 创造力下降 10-20% | **警告**，迭代优化 |

### 3.4 评委模型

- **Qwen-Max**：中文评估首选
- **GPT-4**：英文评估、交叉验证
- **DeepSeek-R1**：快速批量评估
- **人工**：Layer 3 验收、争议仲裁

## 4. 第 10 章：基础设施与运维（Infra / Ops）· 已定要点

### 4.1 数据流架构

- **产品数据流**：端侧推理（实时）→ 定稿确认 → 云端存储（持久化）
- **训练数据流**：定稿卡片 → 用户授权 → 训练候选池 → 周期性训练
- **控制点**：用户数据进训练池必须显式授权，授权可撤销

### 4.2 数据状态流转

```
草稿 → 定稿 → 产品库（全用户）
              ↓ 用户授权
         训练候选池 → 训练完成 → 新模型版本 → 推送更新
```

### 4.3 用户隐私开关

| 设计原则 | 说明 |
|----------|------|
| 默认保守 | 新用户默认「不参与模型改进」 |
| 显式授权 | 参与训练需用户主动开启 |
| 即时生效 | 开关变更立即生效 |
| 可逆可删 | 可随时关闭，可请求删除已贡献数据 |

### 4.4 平台规范分层

| 层级 | 类型 | 示例 |
|------|------|------|
| **L1** | 法律法规 | GDPR、个人信息保护法、生成式AI管理 |
| **L2** | 平台政策 | Google Play 政策、模型许可证 |
| **L3** | 产品自律 | 内容安全准则、数据使用边界 |
| **L4** | 技术实现 | 安全编码、加密策略、审计日志 |

### 4.5 运维类型

| 类型 | 场景 | 响应时效 |
|------|------|----------|
| 产品运维 | 服务可用性、数据同步 | 小时级 |
| 训练运维 | 训练任务管理、资源调度 | 天级 |
| 安全运维 | 红线事件、安全漏洞 | 分钟级 |
| 数据运维 | 数据清理、合规审计 | 周级 |
