# 7. 数据策略（Data）

> 本文档为项目 **shaping** 阶段方案描述：界定微调数据集配方、评估基准（Benchmark）构建、数据翻译与增强策略。**不包含**具体数据预处理代码、翻译 pipeline 实现或评估脚本。

---

## 7.1 核心数据集

### 7.1.1 主数据集：brainstorm_vicuna_10k

| 属性 | 说明 |
|------|------|
| **来源** | Hugging Face `DevQuasar/brainstorm_vicuna_10k` |
| **规模** | 10k 训练 + 1k 测试 |
| **格式** | 多轮对话（human/gpt 交替），6-24 轮 |
| **主题** | 创意想法、计划制定、问题解决 |
| **语言** | 英文 |
| **质量** | 合成数据，追问自然、展开合理 |
| **契合度** | ⭐⭐⭐⭐⭐ 非常契合「头脑风暴」场景 |

### 7.1.2 对话模式分析

典型对话结构：
1. 用户提出想法（"I want to..." / "I'm thinking of..."）
2. 助手追问澄清（"What kind of..." / "What inspired you..."）
3. 多轮深入后收敛到具体方案

## 7.2 数据增强：中英双语构建

### 7.2.1 翻译策略

用云端 Qwen-Max / Qwen-Plus 将 brainstorm_vicuna_10k 翻译成中文，构建双语平行数据集。

**翻译 Prompt**：
```text
请将以下英文头脑风暴对话翻译成中文。
要求：
1. 保持对话的自然流畅
2. 保留追问和发散的语气
3. 人名、地名可适当保留或音译
4. 输出格式与原数据一致（human/gpt 交替）

原文：
[英文对话]
```

### 7.2.2 质量控制

| 步骤 | 操作 | 通过标准 |
|------|------|----------|
| 机器翻译 | Qwen-Max 批量翻译 | 完成全部 10k 条 |
| 人工抽检 | 随机抽取 100 条检查 | 不自然率 < 10% |
| 修正重翻 | 对质量差的重翻或人工修正 | 修正后通过 |

### 7.2.3 风险预案

- 若翻译质量不佳（> 20% 不自然）→ 换用 DeepSeek 或 GPT-4 重翻
- 若中英文语气差异大 → 保留英文原版为主，中文作为补充而非 1:1

## 7.3 微调数据配方（首版保守）

### 7.3.1 数据配比

| 数据集 | 条数 | 占比 | 说明 |
|--------|------|------|------|
| brainstorm_vicuna_10k（英文原版） | 5,000 | 35% | 核心脑暴能力 |
| brainstorm_vicuna_10k（Qwen 翻译中文版） | 5,000 | 35% | 中文脑暴能力 |
| Alpaca / ShareGPT（中英混合通用） | 3,000 | 25% | 通用能力保底，防遗忘 |
| 自建种子数据（个人创意案例） | 500 | 5% | 个性化风格预留 |
| **总计** | **13,500** | **100%** | |

### 7.3.2 保守原则

- **LoRA 优先**：只训练 1-5% 参数，冻结原模型主体
- **学习率控制**：1e-4（较小值），宁可欠拟合不要过拟合
- **轮数限制**：3 epochs，观察后再决定是否增加
- **混合保底**：通用数据占比 25%，防止只学会脑暴忘记正常对话

## 7.4 评估基准（Benchmark）

### 7.4.1 机器评估方案（LLM-as-a-Judge）

采用机器自动评估替代人工打分，确保可评估大规模数据集（2000+条）。

**评委模型选择**：

| 评委模型 | 优势 | 成本 | 推荐场景 |
|----------|------|------|----------|
| **Qwen-Max** | 中文理解强、对齐云端选择 | 阿里云按token计费 | 中文评估首选 |
| **GPT-4** | 英文强、评判经验丰富 | OpenAI API | 英文评估、交叉验证 |
| **DeepSeek-R1** | 性价比高、推理过程可见 | 便宜 | 快速批量评估 |

**评分维度（1-5 分制）**：

| 维度 | 含义 | 5分标准 |
|------|------|---------|
| **相关性** | 回答是否紧扣问题 | 完全理解意图，无跑题 |
| **连贯性** | 多轮对话是否流畅 | 上下文一致，过渡自然 |
| **有用性** | 是否提供有价值的追问/建议 | 追问有深度，建议可执行 |
| **创造性** | 是否提供新颖角度或思路 | 脑暴场景专用，发散合理 |

### 7.4.2 公开基准数据集（直接使用）

从 Hugging Face 选用已验证的评估基准：

| 数据集 | 规模 | 语言 | 特点 | 评估内容 |
|--------|------|------|------|----------|
| **X-AlpacaEval** | 805 条 | 中英等5语言 | AlpacaEval专业翻译版 | 通用指令遵循 |
| **CMT-Eval** | 596 条对话（4,431轮） | 中文 | Speech Act框架 | 中文多轮对话 |
| **brainstorm_vicuna_10k 测试集** | 1,000 条 | 英文 | 原数据集自带 | 脑暴专项 |
| **brainstorm-v3.1_vicnua_1k** | 1,000 条 | 英文 | 带Markdown小结版 | 脑暴+总结提炼 |
| **brainstorm-v2.1_vicuna_1k** | 1,000 条 | 英文 | 追问深化版 | 脑暴追问深度 |
| **MT-Bench** | 80 条 | 英文 | 多轮深度对话 | 多轮能力 |

**公开基准总计**：约 4,500 条（统计显著）

**数据来源**：
- X-AlpacaEval: https://huggingface.co/datasets/zhihz0535/X-AlpacaEval
- CMT-Eval: https://aclanthology.org/2025.findings-emnlp.992/（需确认 Hugging Face 发布情况）
- brainstorm_vicuna_10k: 自带 test split 1k 条
- brainstorm-v3.1_vicnua_1k: https://huggingface.co/datasets/DevQuasar/brainstorm-v3.1_vicnua_1k（带小结，适合评估总结能力）
- brainstorm-v2.1_vicuna_1k: https://huggingface.co/datasets/DevQuasar/brainstorm-v2.1_vicuna_1k（追问深化版）
- MT-Bench: https://huggingface.co/datasets/yzygalaxy/mt_bench_human_judgments

### 7.4.3 自建补充评估集

弥补公开基准未覆盖的领域：

| 领域 | 数量 | 来源 | 用途 |
|------|------|------|------|
| 创意写作 | 100 条 | 从 creative_writing/ember-dataset 抽取 | 创意生成评估 |
| 灵感卡片总结 | 50 条 | 自建：长对话→结构化卡片 | 产品核心功能评估 |
| 产品特定场景 | 100 条 | 自建：针对应用场景设计 | 产品对齐评估 |

**自建总计**：250 条

**评估集总规模**：约 4,750 条（足以检测 > 3% 的能力变化，统计学高度显著）

### 7.4.4 执行流程

```
阶段1：基线测试
    ↓
加载基座模型（Gemma-4-E2B-IT 或 Qwen3.5-2B）
    ↓
跑公开基准集（2,500条）+ 自建集（250条）
    ↓
LLM评委（Qwen-Max）自动打分
    ↓
记录基线分数（均值、标准差、分维度）

阶段2：微调后测试
    ↓
加载微调后 LoRA 模型
    ↓
跑相同评估数据集（2,750条）
    ↓
LLM评委自动打分
    ↓
记录新分数

阶段3：对比分析
    ↓
统计对比：均值差异、t-test显著性检验
    ↓
分维度报告：脑暴/通用/中文/英文各能力变化
    ↓
止损决策：是否接受微调结果
```

### 7.4.5 评估Prompt模板（示例）

```text
你是一位专业的对话质量评估专家。请对以下AI助手的回答进行评估。

【评估维度】（1-5分）
1. 相关性：回答是否紧扣用户问题，是否理解用户意图
2. 连贯性：对话是否流畅自然，上下文是否一致  
3. 有用性：是否提供有价值的追问、建议或信息
4. 创造性：是否提供新颖的角度或思路（脑暴场景）

【对话历史】
{conversation_history}

【当前问题】
{user_input}

【助手回答】
{assistant_output}

请输出JSON格式评分：
{
  "relevance": 4,
  "coherence": 4,
  "helpfulness": 5,
  "creativity": 4,
  "comment": "追问有深度，建议具体可执行，但可进一步发散"
}
```

### 7.4.6 止损线与决策标准

| 情况 | 统计标准 | 决策 |
|------|----------|------|
| 任一维度显著下降 | p < 0.05 且下降 > 10% | 警告，分析原因 |
| 任一维度大幅下降 | p < 0.01 且下降 > 20% | 回退到基座模型 |
| 脑暴能力不升反降 | 上升 < 5% 或下降 | 调整数据配方重训 |
| 中文能力显著下降 | p < 0.05 且下降 > 15% | 增加中文数据或换 Qwen3.5 |

**统计方法**：配对 t-test 比较微调前后同一样本的评分差异。

## 7.5 第二阶段数据（个性化微调）

### 7.5.1 数据来源

- 用户在应用中产生的**定稿卡片**（高信号）
- 用户标记"有用"的脑暴对话轮次
- 经用户**显式授权**后进入训练池

### 7.5.2 触发条件（shaping 占位）

不定具体数值，概念层保留以下选项：
- 累积 N 条定稿卡片后触发
- 每两周周期性触发
- 用户显式点击"优化我的模型"触发

### 7.5.3 数据质量过滤

- 草稿状态内容**默认不进**训练池
- 用户删除或撤回的内容**立即从**训练池移除
- 定期人工抽检用户数据质量
